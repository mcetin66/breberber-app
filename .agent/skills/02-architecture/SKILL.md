# Architecture - Sistem Tasarımı

> Sistem mimarisi, database seçimi, scaling stratejisi ve mimari pattern'ler için.

## Ne Zaman Kullanılmalı?

| Senaryo | Örnek |
|---------|-------|
| Yeni sistem tasarımı | "E-ticaret platformu mimarisi" |
| Microservices kararı | "Hangi servislere bölmeliyiz?" |
| Database seçimi | "PostgreSQL mı MongoDB mi?" |
| Scaling stratejisi | "10x kullanıcıya nasıl ölçekleniriz?" |
| Event-driven mimari | "Kafka ile async iletişim" |

## Key Principles

- ADR (Architecture Decision Record) yaz
- Trade-off'ları belgele
- Ölçeklenebilirliği düşün
- Tek sorumluluk prensibi
- Loose coupling, high cohesion
